Colisão de seletores em CSS

Este exemplo ilustra, simplificadamente, um problema decorrente da incorporação de folhas de estilo que estabelecem seletores que atingem os mesmos elementos HTML. A consequência dessa colisão de seletores é a apresentação disfuncional das páginas web, ocasionando quebra de layout.

Mensagem: alerta padrão da aplicação web
Região DIV estilizada com a classe default-alert, definida na folha de estilos app.css

Neste exemplo, a página web carrega uma folha de estilos (app.css) contendo um seletor DIV.default-alert cuja visualização pretendida é ilustrada ao lado. A página incorpora ainda um componente de terceiros denominado my-widget, cujo comportamento causará uma colisão de seletores de CSS.

O componente my-widget inclui um botão que, ao ser acionado, efetua a carga de uma segunda folha de estilos (widget.css). Essa folha de estilos também define um seletor DIV.default-alert, que colide com o seletor originalmente carregado pela página. Isso causa uma mudança inesperada no layout demonstrado à direita.

Técnicas foram desenvolvidas para reduzir a possibilidade de ocorrência de colisões de seletor, como a prefixação de namespaces em seletores de CSS. Porém, uma solução mais robusta é alcançada pelo uso de shadow DOM.
