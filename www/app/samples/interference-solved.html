Solucionando a colisão de seletores

O exercício de demonstração de interferências evidencia a colisão de seletores de CSS causada pelo carregamento de folhas de estilo que definem visualizações diferentes para um mesmo seletor. Neste exemplo, esse problema é mitigado pelo uso de shadow DOM.

Mensagem: alerta padrão da aplicação web
Região DIV estilizada com a classe default-alert, definida na folha de estilos app.css

Como naquele exercício, esta página web carrega uma folha de estilos (app.css) contendo um seletor DIV.default-alert cuja visualização pretendida é ilustrada ao lado. Nesta página também é incorporado um componente de terceiros denominado my-widget, porém seu funcionamento se baseia no recurso de shadow DOM, o que impedirá uma colisão de seletores de CSS.

Isso é possível por causa do isolamento da estrutura de DOM contida na chamada shadow root do componente my-widget. Assim, o carregamento de folha de estilos widget.css se manifestará exclusivamente no componente, sem afetar a folha de estilos global da página. Desta forma, o layout demonstrado à direita não será modificado.
